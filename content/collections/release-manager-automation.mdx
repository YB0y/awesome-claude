## What this collection sets up

A release-manager path from day-to-day commits through green CI to drafted notes
and changelog artifacts. It complements broader maintainer bundles by focusing on
the cut-release sequence rather than general PR queue operations.

## Workflow

1. Use `git-smart-commit` for Conventional Commit messages during the release branch.
2. Run `checkpoint-manager` before risky refactors near release freeze.
3. Clear red CI with `ci-failure-triage` before tagging.
4. Draft grouped notes with `release-notes-drafting`.
5. Apply `git-cliff-release-changelog-capability-pack` when teams prefer git-cliff output.
6. Optionally attach `workflow-completion-report` for session summaries during release week.

## Duplicate Check

Complements `github-maintainer-operations` (broader maintainer index). This
collection focuses narrowly on release-manager automation steps and changelog tooling.
